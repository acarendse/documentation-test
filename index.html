Shared Mailbox API Reference

Micro-service responsible for the shared mailbox functionality.

API Endpoint
https://virtserver.swaggerhub.com/acarendse/sharedmailbox-osb/1.0.0
Contact: acarendse@sun.ac.za
Request Content-Types: application/json
Response Content-Types: application/json
Schemes: https, http
Version: 1.0.0

Paths

mailbox

POST /api/mailbox

This service will be used to create a shared mailbox, with all the associated users.

sharedMailBoxRequestPayload

The payload to add the shared mailbox

Request Content-Types: application/json
Request Example
{
  "name": "TESTMAILBOX",
  "display_name": "TESTMAILBOX_DISPLAY",
  "owner": "2355855",
  "full_access": [
    "string"
  ],
  "send_as_access": [
    "string"
  ],
  "send_on_behalf_of_access": [
    "string"
  ]
}
201 Created
genericStatusResponsePayload

Success response if the mailbox was added..

400 Bad Request
validationErrorResponse

This will usually be for validation errors, that that can be fixed by the client.

default
serverErrorResponse

Unexpected error

Response Content-Types: application/json
Response Example (201 Created) 
{
  "status": "string"
}
Response Example (400 Bad Request) 
{
  "errorCode": "FORM_DATA_INVALID",
  "businessMessage": "The username field needs to be set.",
  "technicalMessage": "string"
}
Response Example (default ) 
{
  "errorCode": "string",
  "businessMessage": "string",
  "technicalMessage": "Some stack trace."
}

Schema Definitions

genericStatusResponsePayload: object

status: string SUCCESS, FAILURE

If the request was successfull or not.

Example
{
  "status": "string"
}

sharedMailBoxRequestPayload: object

name: string

The Shared mailbox name.

display_name: string

The Shared mailbox display name.

owner: string

The owner's us number

full_access: string[]

The us number's of the users that have full access to the mailbox.

send_as_access: string[]

The us number's of the users that have send as access to the mailbox.

send_on_behalf_of_access: string[]

The us number's of the users that have send on behalf of access to the mailbox.

Example
{
  "name": "TESTMAILBOX",
  "display_name": "TESTMAILBOX_DISPLAY",
  "owner": "2355855",
  "full_access": [
    "string"
  ],
  "send_as_access": [
    "string"
  ],
  "send_on_behalf_of_access": [
    "string"
  ]
}

validationErrorResponse: object

errorCode: string FORM_DATA_INVALID

An error code to identify the type of error.

businessMessage: string

An error message that can potentially be shown to an user.

technicalMessage: string

A stacktrace to help developers debug issues, will mostly be used for 500 internal server errors.

Example
{
  "errorCode": "FORM_DATA_INVALID",
  "businessMessage": "The username field needs to be set.",
  "technicalMessage": "string"
}

serverErrorResponse: object

errorCode: string

An error code to identify the type of error.

businessMessage: string

An error message that can potentially be shown to an user.

technicalMessage: string

A stacktrace to help developers debug issues, will mostly be used for 500 internal server errors.

Example
{
  "errorCode": "string",
  "businessMessage": "string",
  "technicalMessage": "Some stack trace."
}
